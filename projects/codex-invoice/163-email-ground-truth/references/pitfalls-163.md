# 163 邮箱已知陷阱

## 陷阱 1：IMAP ID 命令

**现象**：登录成功但后续 SEARCH/FETCH 命令返回 BAD 或无响应
**原因**：163/网易要求登录后发送 RFC 2971 ID 命令
**修复**：在 login() 之后立即发送 ID 命令（参见 workflow-163.md）

## 陷阱 2：行程报销单被误判为打车发票

**现象**：`滴滴出行行程报销单*.pdf` 被标记为 `打车发票` 而非 `打车行程单`
**原因**：build_truth_dataset.py 的类型判定逻辑未正确区分发票和行程单
**判定规则**：
- 文件名含"行程报销单"或"行程单" + 无 invoice_number → `打车行程单`
- 文件名含"电子发票" + 有 invoice_number → `打车发票`

## 陷阱 3：联通通讯费被排除

**现象**：4张联通发票被标记为 `其他` → `EXCLUDED_OUT_OF_SCOPE_TYPE`
**原因**：seller 字段是"中国联合网络通信有限公司泰安市分公司"，不含"联通"
**修复**：用 `联合网络` 而非 `联通` 匹配

## 陷阱 4：转发邮件结构

QQ 转发到 163 的邮件保留原始附件，但邮件头信息会变化：
- From 变为 `<your-qq-email>@qq.com`
- Subject 前缀加 `Fwd:`
- Date 变为转发时间而非原始时间

## 陷阱 5：163 邮箱文件夹 UTF-7 编码

163 的文件夹名不是明文：
- `&g0l6P3ux-` = 草稿箱
- 需要用 `INBOX` 作为固定文件夹名
