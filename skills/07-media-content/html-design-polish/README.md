# HTML Design Polish

面向网页、内部工具、导航页与产品页面的 HTML 美化 Skill。它通过产品模型、用户任务、信息架构、响应式布局和视觉系统的顺序诊断与执行，提升页面的清晰度、层级和品牌一致性。

## 适用场景

- 产品页、内部工具、导航中心与移动网页的视觉优化或重设计。
- 页面信息层级不清、卡片权重相同、界面显得模板化或缺少品牌感。
- 需要基于参考图提炼层级、材质、对比、构图和交互语气，而非逐像素复刻。
- 需要在不破坏链接、路由、数据、业务规则和交互状态的前提下改善 HTML 页面。

## 快速开始

在请求中说明页面类型、目标用户、需要保留的行为，以及希望优化或参考的方向：

```text
使用 html-design-polish 美化这个内部导航页。保留所有现有链接、服务状态和卡片内容；
参考 assets 中的深色高对比导航风格，重点改善首屏层级、卡片分组和移动端布局。
```

执行前请阅读 [SKILL.md](SKILL.md) 的完整规则。该文件是本目录唯一的 Agent 规则文件。

## 视觉参考

`assets/` 中的三张导航页截图分别体现：

- 深色网格与冷色发光材质下的服务导航层级。
- 渐变城市背景、半透明卡片与多产品入口的组织方式。
- 深色信息空间中，以大标题、状态标签和产品卡片建立清晰优先级的方式。

它们仅用于提取视觉 DNA；不要复刻文字、数据、图标、品牌名称、布局或页面功能。

## 内容边界

- `docs/`：说明性参考材料。
- `examples/`：可复用的提示词、前后对比或示例项目。
- `assets/`：视觉参考、流程图与截图资源。
- 不要提交凭据、访问令牌、部署状态或真实业务数据。
