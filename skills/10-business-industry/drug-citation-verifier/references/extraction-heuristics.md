## 3. 上下文关键词(`DRUG_CONTEXT_KEYWORDS`)

用于关联药名与上下文(例如,识别"洛拉替尼"后面"商品名:博瑞纳"中的商品名)。

```
商品名 / 通用名 / 原研 / 仿制 / 剂量 / 推荐剂量 / 治疗
I 级推荐 / Ⅰ级推荐 / II 级推荐 / Ⅱ级推荐
首选 / 一线 / 二线 / 批准 / 获批 / 上市
```

## 4. 长度限制

| 限制 | 值 |
|-----|---|
| 中文药名最小长度 | 3 字 |
| 中文药名最大长度 | 12 字 |
| 英文药名最小长度 | 5 字符 |
| 英文药名最大长度 | 30 字符 |
| 商品名括号内最大长度 | 40 字符 |
| 上下文窗口 | ±60 字符 |

## 5. 关键边界

### 5.1 这是命名学,不是药品库

✅ 允许:`DRUG_NAME_SUFFIXES_CN = ("替尼", "单抗", ...)`
- 这是"中文药名的语法",任何符合后缀的字符串都视为疑似药名
- 不是"哪些药存在"

❌ 禁止:`KNOWN_DRUGS = {"<某药>": "<某英文名>", ...}`
- 这是"已知药品列表",违反 P0
- 一旦允许,会演化成全药品库

### 5.2 抽取后必须核对源

抽取出的"疑似药名"**仅是疑似**。必须由 verifier 拿去 A11
`verify_claim_against_source` 核对源:

- 源里有此药名 → 真药
- 源里没此药名 → LLM 编造 / 不属于本指南

### 5.3 商品名学习自源

报告里的商品名(如"博瑞纳")也走相同流程:
- 抽取时一并记录(`brand_in_context`)
- 核对时作为额外 keyword
- 源里若没出现 → critical
