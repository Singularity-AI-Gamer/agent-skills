# 隐私与脱敏说明

本仓库公开，所有技能在纳入仓库前已做脱敏扫描，原则如下。

## 通用脱敏规则

发布到仓库的任何文件，都不得包含以下内容：

| 类别 | 处理方式 |
|---|---|
| 邮箱地址 | 用 `<your-email>@example.com` 占位 |
| 手机号 / 联系方式 | 用 `<your-phone>` 占位 |
| API Key / Token / Secret / 授权码 | 用 `<YOUR_*_TOKEN>` / `<YOUR_*_KEY>` 占位 |
| Access Key ID / Secret | 用 `<YOUR_ACCESS_KEY>` 占位 |
| 已签名的 URL（含 AKID、签名） | 禁止入库（见 `.gitignore`） |
| 个人昵称 / 真名 | 替换为 `用户` 或 `Anon` |
| 本地用户目录（如 `C:\Users\<name>\...`） | 改写为 `%USERPROFILE%` 或 `~` |
| 公司内部工具名 / 产品代号 | 改写为通用描述 |

## 扫描与拦截

- `.gitignore` 已配置常见密钥文件模式（`.env`、`*.key`、`*.pem`、`*_token*` 等）。
- GitHub 仓库启用 Push Protection，提交含明文密钥时会被自动拦截。
- 新增技能前建议本地跑一次 `git diff` 或 `grep` 复查。

## 使用者须知

从本仓库拉取技能后，需要自行填入凭据的地方会以 `<YOUR_*>` 或 `<your-*>` 占位符标记。
请勿将填入真实凭据的文件再提交回仓库。

## 发现泄漏时的处理

若发现已合入的历史提交包含真实凭据：

1. 立即在对应服务中轮换（rotate）该凭据。
2. 从工作区移除（`git rm`）并重写索引。
3. 如需擦除历史，使用 `git filter-repo` 或等效工具，然后强推。
4. 公开仓库被 clone 后历史无法回收，务必以"先轮换"为第一步。
